# Структури та підходи до сховища знань

Огляд популярних структур для нотатків — оберіть ту, що найкраще підходить вам, або комбінуйте їх.

1) PARA (Projects, Areas, Resources, Archives)
+- Projects: речі з ціллю та дедлайном.
+- Areas: довготривалі відповыдальності (здоров'я, робота, фінанси).
+- Resources: довідкові матеріали, посилання, книги.
+- Archives: завершені проекти й старі нотатки.

Коли підходить: для керування роботою й особистими зонами відповідальності.

2) Zettelkasten (номеровані атомарні нотатки)
+- Атомні нотатки з унікальними ідентифікаторами, багато внутрішніх посилань.
+- Добре підходить для дослідження, написання та побудови знань з часом.

3) MOC (Map of Content)
+- Централізовані «карти» теми або проєкту, що агрегують посилання на важливі нотатки.

4) Atomic Notes / Evergreen Notes
+- Маленькі, незалежні, редаговані з часом нотатки з чіткою ідеєю в одній нотатці.

Комбіноване використання
+- Багато користувачів поєднують PARA (для організації) + Zettelkasten/Atomic (для знань і ідей) + MOC (для навігації).

Рекомендований робочий процес (щоденний):
+- Capture → Process (Inbox) → Organize (Projects/Areas/Resources) → Review (щотижнево/щомісяця).

Далі в шаблонах — щоденна нотатка та інструменти для обробки Inbox.
